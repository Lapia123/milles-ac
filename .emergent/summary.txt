<analysis>**original_problem_statement:**
The user wants to build a back-office accounting software for an FX (foreign exchange) broker.

The core requirements have evolved over several iterations:
- **PRODUCT REQUIREMENTS:**
    - **User Roles:** Admin, Sub-admin, and Accountant for the back office.
    - **Authentication:** JWT (email/password) and Google Social Login.
    - **Client Management:** Create, Read, Update, Delete (CRUD) for clients, including KYC status, MT5 Number, and CRM Customer ID.
    - **Transaction Management:**
        - Track deposits, withdrawals, and transfers.
        - Transactions are linked directly to a client.
        - Must support uploading a screenshot or proof of payment.
        - Must specify a destination (e.g., a Treasury/Bank account or a PSP).
        - Multi-currency support for transactions (e.g., in AED), with all final balances tracked in USD.
    - **Treasury Management:** An area to manage bank accounts in different currencies. The total treasury balance is shown in USD.
    - **Accountant Dashboard:** A dedicated view for accountants to approve or reject pending transactions. Approvals are protected by a simple mathematical captcha (addition only).
    - **PSP (Payment Service Provider) Management:**
        - Ability to add and manage PSPs.
        - Track PSP commissions, settlement times, and settlement history.
        - Use PSPs as a destination for client deposits.
    - **Reporting:** Basic account balances and transaction history. (Advanced reporting like P&L was initially requested but has not been prioritized yet).

**User's preferred language**: English

**what currently exists?**
A full-stack web application with a React frontend and FastAPI backend.
- **Backend:** A single  file contains all API endpoints and database models for users, clients, transactions, treasury accounts, and PSPs. It includes logic for JWT/Google auth, multi-currency conversion (using hardcoded rates), and the transaction approval workflow.
- **Frontend:** A React SPA with pages for Login, Dashboard, Client Management, Transaction Management, Treasury Management, Accountant Approvals, and User Settings. The UI is built with TailwindCSS and Shadcn/UI components.
- **Database:** MongoDB is used with collections for users, clients, transactions, treasury_accounts, and psps.
